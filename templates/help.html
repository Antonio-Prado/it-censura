Panoramica

Il DNS Blacklist Manager è uno strumento web per gestire le blacklist del server DNS ricorsivo Unbound su FreeBSD, integrato con il sistema censorship. Permette di verificare se un dominio è bloccato, aggiungere o rimuovere voci dalla blacklist manuale, aggiornare le liste dai sorgenti remoti e ricaricare il servizio DNS.

Le blacklist sono divise in due categorie: ufficiali (disposizioni di legge italiane) e community (liste pubbliche aggiornate automaticamente). Solo la lista manual è modificabile dalla GUI.

Ricerca di un dominio

🔍
Come cercare

Inserisci un nome di dominio nel campo in alto (es. example.com) e premi Cerca o il tasto Invio.

🚫
Dominio trovato

Il banner diventa rosso e mostra le liste in cui il dominio è presente, con tag colorati per categoria. Se è nella lista manuale appare il pulsante per rimuoverlo.

✓
Dominio non trovato

Il banner diventa verde. Appare il pulsante per aggiungerlo alla blacklist manuale.

Nota: La ricerca viene eseguita su tutti i file di blacklist presenti sul server. Sui file più grandi (es. porn, malware) può richiedere alcuni secondi. Lo spinner sul pulsante indica che la ricerca è in corso.

Gestione blacklist manuale

La blacklist manuale (blacklist_manual.txt) è l'unica lista modificabile direttamente. Le altre liste vengono aggiornate dagli script automatici di censorship.

+
Aggiungere un dominio

Scrivi il dominio nel campo in alto a destra (es. sito-da-bloccare.com) e premi + o Invio. Il dominio appare subito nella lista. In alternativa, dopo una ricerca con esito negativo, usa il pulsante "Aggiungi alla blacklist manuale".

✕
Rimuovere un dominio

Clicca la ✕ accanto al dominio nella lista laterale, oppure dopo una ricerca con esito positivo usa il pulsante "Rimuovi dalla blacklist manuale". Verrà chiesta conferma prima della rimozione.

Attenzione: Dopo ogni aggiunta o rimozione, il server rigenera automaticamente i file di configurazione Unbound. Questa operazione richiede circa 2 minuti (script BL_concat.sh). Il dominio è già visibile nella lista, ma Unbound non applicherà il blocco finché non si ricarica il servizio (vedi sotto).

Formato dei domini accettati

Sono accettati solo nomi di dominio validi, es. example.com, sub.example.co.uk. Non sono accettati: URL completi (https://…), indirizzi IP, domini senza TLD (localhost), stringhe con caratteri non validi.

Ricarica del servizio Unbound

Dopo aver modificato la blacklist manuale (e atteso la rigenerazione dei file), è necessario ricaricare Unbound affinché le modifiche abbiano effetto sulle query DNS.

↺
Pulsante "Ricarica Unbound"

Esegue service unbound reload sul server. L'operazione è immediata. Attendere che il server risponda con il messaggio di conferma prima di procedere.

Procedura completa per bloccare un dominio:
1. Aggiungilo alla blacklist manuale
2. Attendi ~2 minuti (rigenerazione configurazione)
3. Clicca Ricarica Unbound
4. Verifica con host dominio.com 127.0.0.1 dal server

Aggiornamento blacklist

Le blacklist vengono aggiornate quotidianamente tramite script automatici. È possibile forzare un aggiornamento manuale direttamente dalla GUI.

↓
Pulsante "Aggiorna liste"

Avvia lo script update_blacklists.sh che scarica e rigenera tutte le liste dai sorgenti remoti (liste ufficiali e community). L'output dello script viene mostrato in tempo reale nel log sottostante.

⏱
Durata

L'operazione richiede diversi minuti: il download delle liste più grandi (es. porn, malware) e la successiva concatenazione (BL_concat.sh, ~2 min) sono le fasi più lente. La pagina può essere lasciata aperta: se viene ricaricata, il polling riprende automaticamente.

✓
Al termine

Le statistiche vengono ricalcolate automaticamente. Clicca poi Ricarica Unbound per applicare le nuove liste al DNS.

Attenzione: Non avviare un aggiornamento mentre è già in corso un altro. Il pulsante viene disabilitato automaticamente durante l'esecuzione.

Statistiche blacklist

Il pannello statistiche mostra il numero di voci presenti in ciascuna lista. Al primo avvio o dopo un aggiornamento, il calcolo viene eseguito in background e può richiedere qualche decina di secondi: durante questo tempo viene mostrato lo spinner "Calcolo in corso…". Una volta calcolate, le statistiche vengono mantenute in cache per 5 minuti.

Il pulsante Aggiorna forza il ricalcolo immediato.
